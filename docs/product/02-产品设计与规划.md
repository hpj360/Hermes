## 一、产品定位

### 1.1 一句话定位

**面向个人知识工作者与小团队的、Agent 驱动的 AI 原生知识库平台。**

### 1.2 核心理念

> 知识库不是更智能的收藏夹，而是 Agent 做事时调用的业务知识层。

将传统知识库的"存储-分类-检索"模式升级为：
```
原始资料层  →  业务知识层  →  场景调用层
（多源接入）  （Agent 抽取/关联/索引）  （写作/研究/产品/服务）
```

### 1.3 差异化卖点（USP）

| USP | 说明 |
|-----|------|
| **Agent 业务知识层** | 知识可被 Agent 稳定调用，而非仅用于检索 |
| **三层记忆模型** | L1 工作 / L2 情节 / L3 语义，业界领先 |
| **本地优先 + 多 LLM 路由** | 隐私安全 + 成本可控（14 家 Provider） |
| **Skill 工程化** | 30 个已沉淀 skills 可编排为知识工作流 |
| **RAG + Agent 双引擎** | 检索增强 + 自主执行 |

### 1.4 不做什么（边界）

- ❌ 不做协作套件（不和飞书/Notion 正面竞争富文本协作）
- ❌ 不做企业级 ERP/OA 集成（避开企业级复杂权限与流程）
- ❌ 不做通用 Agent 操作系统（聚焦知识库垂直场景）
- ❌ 不做大规模云服务（首期坚持本地优先/自托管）

## 二、目标用户与场景

### 2.1 核心用户画像

| 用户类型 | 占比 | 典型画像 | 核心痛点 |
|----------|:----:|----------|----------|
| **个人知识工作者** | 40% | 内容创作者、研究者、独立 PM | 资料散落、写作提效、知识沉淀 |
| **小团队/独立开发者** | 30% | 3-10 人技术团队 | 团队知识库、私有部署、自动化 |
| **AI 从业者/研究者** | 20% | Agent 开发者、RAG 研究者 | 记忆模型实验、模型对比 |
| **企业创新团队** | 10% | 大企业内部创新/PoC 团队 | 私有化、与现有系统集成 |

### 2.2 核心使用场景

#### 场景一：个人内容创作者（如小红书博主）
- **痛点**：选题灵感散落、AI 写作每次都要重新解释背景
- **方案**：导入历史内容 → Agent 抽取风格/选题/标签 → AI 写作时调用个人风格
- **价值**：风格稳定、选题复用、写作效率提升 3x

#### 场景二：技术研究小团队
- **痛点**：技术文档散落 GitHub/Notion/语雀，新人 onboarding 慢
- **方案**：多源接入 → 统一向量化 → Agent 智能问答（带引用）
- **价值**：新人 1 周上手变 2 天，技术决策有依据

#### 场景三：Agent 开发者
- **痛点**：每次搭 RAG 都从零开始，记忆模型难复现
- **方案**：复用 Hermes 三层记忆 + RAG 引擎，专注业务逻辑
- **价值**：原型搭建从 1 周变 1 天

#### 场景四：企业内部 PoC
- **痛点**：公有云 AI 不合规，自建成本高
- **方案**：本地部署 + 私有 LLM（Ollama）+ 内部文档接入
- **价值**：1 周内跑通 PoC，数据不出域

## 三、产品功能架构

### 3.1 总体架构（四层）

```
┌─────────────────────────────────────────────────────┐
│            应用层（Application Layer）                │
│  Web UI · CLI · API · MCP Server · Skill 编排        │
├─────────────────────────────────────────────────────┤
│            智能层（Intelligence Layer）               │
│  RAG 引擎 · Agent 编排 · 三层记忆 · 多 LLM 路由       │
├─────────────────────────────────────────────────────┤
│            知识层（Knowledge Layer）                  │
│  业务知识抽取 · 实体/关系 · 索引/检索 · 引用追溯      │
├─────────────────────────────────────────────────────┤
│            数据层（Data Layer）                       │
│  原始资料 · 向量库 · 关系库 · 全文索引 · 文件存储     │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心功能模块

#### 模块一：知识接入（Ingestion）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| 文件导入 | PDF / Word / Markdown / TXT / Excel / 图片 | P0 |
| 网页抓取 | URL 抓取 + 正文提取（含 wechat-reader / youtube-watcher / douyin-reader 复用） | P0 |
| API 接入 | Notion / GitHub / Obsidian / Trello（已有 skills） | P1 |
| 增量同步 | 文件变更自动重新向量化（避免僵尸知识） | P0 |
| 多模态解析 | OCR / 语音转写 / 视频帧抽取 | P2 |

#### 模块二：知识处理（Processing）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| 文档解析 | Unstructured 统一解析多格式 | P0 |
| 智能分片 | 按章节/段落/句子分片，支持重叠 | P0 |
| 向量化 | bge-m3（本地）/ OpenAI embedding（云端） | P0 |
| 实体抽取 | Agent 抽取概念/实体/关系，构建业务知识层 | P1 |
| 元数据管理 | 来源/时间/作者/标签/版本 | P0 |

#### 模块三：知识检索（Retrieval）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| 语义检索 | 向量相似度搜索 | P0 |
| 关键词检索 | BM25 全文检索 | P0 |
| 混合检索 | BM25 + 向量 + RRF 融合（与 OpenClaw 一致） | P0 |
| 重排序 | bge-reranker 精排 | P1 |
| 时间衰减 | 30 天半衰期（与三层记忆模型一致） | P1 |
| 引用追溯 | 每条答案可追溯到原文档/分片 | P0 |

#### 模块四：Agent 智能层（Agent）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| 三层记忆 | L1 工作 / L2 情节 / L3 语义（已有） | P0 |
| Agent 编排 | 基于已有 skills 的工作流编排 | P1 |
| 任务契约 | 结构化任务定义，避免幻觉累积 | P0 |
| 多模型路由 | 简单任务用小模型，复杂任务用大模型 | P0 |
| 流式响应 | 流式输出降低等待焦虑 | P0 |
| 人工兜底 | 高风险任务转人工审核 | P1 |

#### 模块五：知识应用（Application）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| 智能问答 | 基于知识库的 RAG 问答（带引用） | P0 |
| AI 写作 | 调用个人风格/历史知识的写作辅助 | P0 |
| 知识图谱 | 可视化实体关系（基于 L3 语义记忆） | P2 |
| 自动摘要 | 文档/会议/对话摘要 | P1 |
| 翻译/改写 | 多语言支持 | P2 |

#### 模块六：协作与管理（Management）

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| Web UI | 文档浏览/问答/管理界面 | P0 |
| CLI | 复用 hermes 命令行（已有） | P0 |
| 权限管理 | 用户/空间/文档级权限 | P1 |
| 版本管理 | 文档版本历史与回滚 | P1 |
| 反馈闭环 | 答案"有用/无用"评价 → 优化队列 | P0 |
| 持续优化 | self-improving-agent + 评估器 | P1 |

## 四、技术架构

### 4.1 技术栈选型

| 层 | 选型 | 理由 |
|----|------|------|
| 后端框架 | FastAPI（Python） | 与 Hermes 同栈，异步性能好 |
| 前端 | React + Vite + TailwindCSS | 主流、轻量、易部署 |
| 数据库 | SQLite（默认）/ PostgreSQL（团队） | 渐进式扩展 |
| 向量库 | sqlite-vec（轻量）/ Qdrant（生产） | 与三层记忆模型一致 |
| 全文索引 | SQLite FTS5 / Elasticsearch（可选） | 内置即可满足 |
| RAG 框架 | LangChain | Python 生态最成熟 |
| 嵌入模型 | bge-m3（本地）/ OpenAI（云端） | 中文优秀、可本地 |
| 重排序 | bge-reranker-base | 本地、轻量 |
| LLM 路由 | Hermes 已有 14 家 Provider | 直接复用 |
| 部署 | Docker Compose | 一键自托管 |

### 4.2 与 Hermes 现有架构的复用

| Hermes 现有资产 | 知识库平台复用方式 |
|-----------------|-------------------|
| `src/hermes/config.py` | 复用 LLM Provider 配置 |
| `src/hermes/skills.py` | 复用 skills 发现与管理 |
| `src/hermes/main.py` CLI | 扩展 `hermes kb` 子命令 |
| `knowledge/memory-model.md` | 三层记忆直接复用 |
| `skills/` 30 个 skills | 内容抓取/读取/搜索直接复用 |
| `manifest.json` | 扩展知识库清单 |

### 4.3 关键数据流

```
[用户/Agent 提问]
       │
       ▼
[意图识别] ──→ [三层记忆检索：L3 永久 + L2 历史]
       │
       ▼
[混合检索：BM25 + 向量 + RRF]
       │
       ▼
[重排序：bge-reranker]
       │
       ▼
[上下文构建 + 引用元数据]
       │
       ▼
[多模型路由：选 LLM]
       │
       ▼
[流式生成 + 引用标注]
       │
       ▼
[反馈记录 → 优化队列]
       │
       ▼
[写入 L2 情节记忆]
```

## 五、产品路线图（Roadmap）

### 5.1 总体节奏

| 阶段 | 时间 | 目标 | 团队规模 |
|------|------|------|:------:|
| **M0：MVP 闭环** | Month 1 | 跑通"导入 → 检索 → 生成 → 引用"最小闭环 | 3-4 人 |
| **M1：Agent 集成** | Month 2-3 | 三层记忆 + Agent 编排 + 多 LLM 路由 | 5-6 人 |
| **M2：体验打磨** | Month 4-5 | Web UI 完善 + 反馈闭环 + 持续优化 | 6-7 人 |
| **M3：场景拓展** | Month 6+ | 多模态 + 知识图谱 + 团队协作 | 7-8 人 |

### 5.2 M0：MVP 闭环（Month 1，4 周）

**目标**：1 个用户能在本地跑通端到端知识库问答

| 周次 | 交付物 | 验收标准 |
|------|--------|----------|
| W1 | 项目骨架 + 数据模型 + Docker 环境 | `docker compose up` 启动成功 |
| W2 | 文件导入 + 解析 + 分片 + 向量化 | 支持 PDF/Markdown/TXT，分片合理 |
| W3 | 混合检索 + RRF + 引用追溯 | Top-5 召回率 ≥80%，答案带引用 |
| W4 | CLI 问答 + 最小 Web UI | 命令行可问答，Web 可浏览/提问 |

**MVP 不做**：多模态、协作、权限、知识图谱、Agent 编排

### 5.3 M1：Agent 集成（Month 2-3，8 周）

**目标**：Hermes 知识库成为 Agent 的业务知识层

| 周次 | 交付物 |
|------|--------|
| W5-6 | 三层记忆集成（L1/L2/L3）+ Memory Flush |
| W7-8 | Agent 编排引擎 + 任务契约 |
| W9-10 | 多 LLM 路由 + 流式响应 + 人工兜底 |
| W11-12 | Skills 工作流编排（复用 30 个 skills） |

### 5.4 M2：体验打磨（Month 4-5，8 周）

**目标**：可对外发布的 v1.0

| 周次 | 交付物 |
|------|--------|
| W13-14 | Web UI 完善（文档管理/问答/设置） |
| W15-16 | 反馈闭环（有用/无用评价 + 优化队列） |
| W17-18 | 持续优化（self-improving-agent + 评估器） |
| W19-20 | 性能优化 + 部署文档 + 用户引导 |

### 5.5 M3：场景拓展（Month 6+）

- 多模态：图片 OCR / 音频转写 / 视频字幕
- 知识图谱：基于 L3 语义记忆的可视化
- 团队协作：多用户 + 权限 + 评论
- 插件市场：开放 Skill / 数据源插件

## 九、立项决策建议

### 9.1 立项建议：**建议立项**

**理由**：
1. Hermes 已有差异化资产（三层记忆 + 30 skills + 多 LLM）
2. AI 原生知识库赛道仍在早期，无绝对头部
3. 本地优先 + Agent 驱动是有清晰差异化的细分市场
4. MVP 闭环 1 个月内可交付，验证成本低

### 9.2 关键前提

- 团队 6 人到位（含 1 名 PM + 1 名 Tech Lead）
- 明确 M0-M1 阶段不做协作、不做企业级集成
- 建立 Beta 用户社区（≥20 人）持续反馈
- 每两周一次产品评审，每月一次 Roadmap 调整

### 9.3 立项后第一周行动项

- [ ] PM：完成 M0 详细需求文档 + 用户访谈大纲
